Overview

This project will be a modular, web based application with a hardware integration to manage, monitor and improve farming practises and calculations of feed demand & cost according to set weight gains of beef cattle. The project will combine tools which are already available as well as new instruments which have not or not optimally been utilised before. It will also enable farmers to upgrade their existing equipment with sensors and user interfaces to make it smarter.

Motivation

With a rapidly growing population and recurring extreme weather events, farming has been getting tougher and more competitive while the demand of food grows, and the accessability of land declines. Many small to medium primary producers are using fertilisers, seeds and pesticides or other resources like fuel for their animals - as they have for decades. New technologies allow us to improve the efficiency and precision in landwork and animal production drastically at reasonable cost. To enable farmers to save money while increasing profit and production output, web based technologies like HTML, JavaScript and databases in combination with hardware which uses GPS, LoRa, weight measurement-systems or other sensors should be used.

Description

This web based solution, which will also be accessable via iOS and Android, will cover the need of the modern farmer from A to Z. For example, the feed calculator for cattle to match a definied weight gain. It will calculate the nutritional demand (energy and protein) depending on the weight, age, weather condition and other circumstances like pregnancy or lactation and find the best feed plan for the animal. While it does these calculation, it will also consider the cost of different products and choose the cheapest option for the farmer while keeping the welfare of the animal in mind.
A second major feature will be the implementation of sensors, actors and displays to save resources and time. Many agricultural implements which are being used by small to medium producers work in a very simple way. A seeder, for instance, has got a little window on the side of the seed box, but does not tell the operator when the box is empty. Optical sensors combined with little graphical user interfaces can not only provide feedback to the operator but even learn about the machine and the output of resources at different settings. The same principle can be used to integrate flow sensors and pressure regulators for sprayer units.
Measuring and calculating is only one part of the system. An intelligent backend will store data and use it to suggest settings or timings. For instance, a weather module which uses external data in the beginning but learns from local weather collected by a hardware module will be able to give the farmer a good outlook for the long term weather. The feed calculator works in a similar way. Different breeds of cattle in different areas convert feed differently. If they don’t perform as expected, the software will either increase or decrease values to adjust to the breed and environment. All those information will be stored locally but synchronised to a cloud. To access the data or to put in more data, little displays can be installed at different places on the farm and in machines. These displays will have various working windows, depending on the task which is being performed. At the grain or hay storage, for instance, the interface will tell the user which quantities are required for which mobs. The scale, in turn, will be able to tell the interface the quantities fed, and adjust the inventory accordingly. The more of these units are used, the more information can be extracted and the more precise and helpful the system will get.
Another feature will be a time tracking system. Especially when different people work on the farm, time tracking can be very valuable. It will allow to keep track of working hours which than helps for payments and it will also allow to record times for different tasks, in different paddocks etc. and therefore help the manager to manage time more efficiently and precisely. The data will be entered via user access to the web interface, app or on the little workstation displays.

Tools and Technologies

On the software side of this project, technologies like HTML, CSS, JavaScript and Databases are vital for the front- and backend. Java is needed for the Android app, Swift is necessary for the iOS app. Additional frameworks like Bootstrap and libraries like jQuery are helpful for the GUI & DOM. The project includes various sensors: optical / ultrasonic sensors, pressure and flow sensors as well as weight sensors and those to identify position, altitude and speed (GNASS). Data in- and output will be realised with Raspberry PIs and Arduinos and compatible touch- and segment displays. The transmission of information can be done by using LoRa components.

Required Skills

To develop the brain & interface for human interaction (the browser based application - front & backend), a solid knowledge in HTML and JavaScript in combination with databases (MongoDB) is required. A good knowledge about jQuery and Bootstrap will accelerate the programming. To utilise smartphones, a solid base in Java and Swift is required. Extensive research as well as experience in farming provide the base for the feed calculation module and the integration of sensors. Mathematics is a major part of the project, however, algebra and elements of curve sketching will be sufficient.
The hardware (sensors, mini computers and displays) is very common and easily to source. At a later stage, the hardware might be replaced by tailor-made products to reduce cost. All the technologies used are widly spread and easily accessible. A lot of them work similar and the knowledge is, with enough time, easily acquirable.

Outcome

If the project is successful, primary producers will reduce the usage of human- and natural resources and therefore:

  • reduce the strain on the environment
    (reduced fertiliser usage and feed wastage, less usage of oils & fuels, diminished or eliminated need to buy new equipment)
  • produce more efficiently
    (smart technologies which 1. reduce time in the paddock and in the office, 2. a measurable and repeatable performance of the animals, 3. maybe an increase of animal capacities)
  • increase the profit
    (surplus production in pasture which can be baled and sold, more production in beef, less cost in resources)

This will enable the producer to be more competitive and to grow with the higher demand.
